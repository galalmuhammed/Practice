Git Commands Cheat Cheet

Important Git Commands

git
Show all git commands.
git -v
Check current Git version.
clear
Clear Terminal.
touch <file-name>
Create a file.
cd <path>
Change directory.
ls
List all items in current folder.
ls -a
List all items including hidden ones.

Configurations

git config --global user.name "Your Name"
Configure name.
git config --global user.email "Your Email"
Configure email.
git config --global init.defaultBranch main
Configure deafult branch.
git config user.name
Check configured username.
git config user.email
Check configured email.
git config init.defaultBranch
Check configured default branch.
git config list
Check all Git configurations.

Working with Git

git init
Initialize a local Git repository.
git status
Check current status of files. (4 status):
Untracked
New file not added yet.
Modified
File changed but not staged.
Staged
File is in staging, ready to commit.
Committed
File saved in history.
git clean -f
Delete all untracked files.
git clean -f <file-name>
Delete a specific untracked file.
git clean -fd
Delete untracked files and directories.
git restore <file-name>
Discard changes and restore file to last commit (tracked files only).
git add <file-name> <other-file-name>
Add one or more files to staging.
git add .
Add all files to staging.
git restore --staged <file-name>
Remove file from staging (back to modified/untracked).
git commit -m "Enter commit message"
Commit staged changes with a message.
git log
Show commit history.
git checkout <commit-id>
Switch to an older commit (detached HEAD).
git checkout <default-branch-name>
Switch back to the latest commit of the branch.
git reset <commit-id>
Reset branch to a specific commit:
git reset --soft <commit-id>
Keep everything staged.
git reset --mixed <commit-id> (default)
Keep changes, but unstaged.
git reset --hard <commit-id>
Erase changes â€” clean working tree. (Dangerous)
git revert <commit-id>
Undo a commit by creating a new one (keeps history):
git revert <commit-id> --no-edit
Use default message; skip editor.
git revert <commit-id> --no-commit
Apply changes only; commit later.

Managing Branches

git branch
List all branches.
git branch <branch-name>
Create a new branch that copies the current one.
git checkout <branch-name>
Switch to a different branch.
git merge <branch-name>
Merge other branch into current one.
git branch -d <branch-name>
Soft delete a local branch. (only if merged)
git branch -D <branch-name>
Hard delete a local branch (forced delete).

Connecting to GitHub

git remote -v
Show remote connections.
git remote add origin <link-for-github-repository>
Connect local repo to GitHub.
git push -u origin <branch-name>
Push branch to GitHub (creates a link to origin).
git push
Works if branch is already linked.
git push --force
Overwrite remote branch.
git push --force-with-lease
Overwrite only if no one else pushed.
git pull -n origin <branch-name>
Pull updates from GitHub.
git pull
Works if branch is already linked.
git clone <repo-URL>
Clone a repository (does 3 things):
Download history
Copies full repo (commits, branches, tags).
Create local repo
Working directory + .git folder.
Set remote
Adds origin pointing to the cloned repo.
